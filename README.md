#  Rendimiento Web 📈

¡Optimizando la Experiencia Nike! ⚡️

Este proyecto se enfoca en evaluar el rendimiento del sitio web de Nike 👟 (nike.com) a través de pruebas de estrés y análisis exhaustivo con Lighthouse. Se buscó identificar áreas de mejora para garantizar una experiencia de usuario óptima tanto en dispositivos de escritorio como móviles.

Tecnologías Clave Utilizadas 🛠️

 * Lighthouse 💡: Herramienta de Google para auditar el rendimiento web, accesibilidad, mejores prácticas y SEO. Se utiliza para generar informes detallados del sitio web de    Nike.
 
 * Análisis Comparativo 📊: Se realizó una comparación entre el rendimiento del sitio web en dispositivos de escritorio y móviles para identificar disparidades y priorizar      áreas de optimización.
   
 * Visualización de Datos 📈: Se crearon gráficos para representar visualmente los resultados de las pruebas de Lighthouse, facilitando la comprensión de las métricas clave.
   
Hallazgos Clave 🔍

* Se identificaron oportunidades de mejora en el rendimiento, especialmente en la versión móvil del sitio web.
* Se evaluó la accesibilidad del sitio web, destacando áreas donde se pueden implementar mejoras para garantizar la inclusión.
* Se analizó el SEO del sitio web, revelando una optimización sólida en general.
* Este análisis proporciona información valiosa para optimizar el sitio web de Nike, mejorando la velocidad de carga, la accesibilidad y la experiencia general del usuario.

# Backend Analysis ⚙️

Desentrañando el Cerebro Digital de Nike 🧠

Este proyecto se sumerge en el análisis del backend del sitio web de Nike (nike.com/us/es/) 🕵️‍♂️, examinando las solicitudes y respuestas del servidor para comprender su funcionamiento interno. El objetivo es identificar posibles áreas de optimización y garantizar una comunicación eficiente entre el cliente y el servidor.

Tecnologías Clave Utilizadas 🛠️

 * Herramientas de Desarrollo del Navegador 🌐: Se utilizaron las herramientas integradas del navegador (Chrome, presumiblemente) para inspeccionar el tráfico de red,      
   incluyendo solicitudes (solicitudes), respuestas (respuestas), encabezados (headers) y códigos de estado HTTP (códigos de estado).
   
 * Análisis de Códigos de Estado HTTP 📊: Se analizaron los códigos de estado HTTP para determinar el éxito o fracaso de las solicitudes, identificando posibles errores o   
   áreas de mejora en la comunicación del servidor.
   
 * Visualización de Datos 📈: Se emplearon cuadros y gráficos para representar la distribución de los códigos de estado HTTP, facilitando la interpretación de los resultados 
   del análisis.
   
Hallazgos Clave 🔍

* Se verificará un alto número de solicitudes exitosas (Estado 200 OK).   
* Se identifican algunos códigos de error (Estado 400, 404, 500) que podrían indicar problemas en el cliente o en el servidor.   
* Se analizó el uso de códigos de estado informativos (204, 304) y de creación (201).   
* Este análisis proporciona una visión valiosa de la arquitectura del backend de Nike, permitiendo identificar áreas de mejora en la eficiencia, confiabilidad y experiencia del usuario.

# Manejo de datos e informes 📊

Garantizando la Calidad en Nike 🚀

Este informe detalla el proceso de pruebas realizadas en el sitio web de Nike, con el objetivo de identificar y documentar posibles defectos o áreas de mejora. Se ejecutaron casos de prueba, se analizaron los resultados y se generaron informes para proporcionar una visión clara del estado de la calidad del sitio.

Elementos Clave Analizados 🔍

 * Ejecución de Casos de Prueba 📝: Se ejecutaron un total de 10 casos de prueba, a incluyendo diferentes funcionalidades del sitio web de Nike.
   
 * Análisis de Resultados 📊: Se analizaron los resultados de los casos de prueba, clasificándolos como "Paso" (éxito) o "Fallo" (defecto). Se calcularon los porcentajes de 
   casos de prueba exitosos y fallidos.
   
 * Identificación de Defectos 🐞: Se identificaron y documentaron los defectos encontrados, incluyendo su tipo (usabilidad) y criticidad (media).
   
 * Priorización de Casos de Prueba ⭐️: Se asignó prioridad a los casos de prueba (alta, media, baja) para enfocar los esfuerzos de prueba en las áreas más críticas.
   
 * Generación de Gráficos 📈: Se crearon gráficos para visualizar los resultados de las pruebas y la prioridad de los casos de prueba, facilitando la comprensión de los 
   datos.
   
Hallazgos Clave 📌

* De los 10 casos de prueba ejecutados, 7 pasaron y 3 fallaron.
* Los defectos encontrados se relacionaron con la usabilidad del sitio web.
* La mayoría de los casos de prueba se clasificaron con prioridad media.
  
Este informe proporciona información valiosa para mejorar la calidad del sitio web de Nike, abordando los defectos identificados y optimizando la experiencia del usuario.

# Bug Reporting 🐞

Cazando Errores en Nike 🕵️‍♀️

Este proyecto se centra en la identificación y documentación de errores 🐞 encontrados en el sitio web de Nike (nike.com/us/es/). Se registraron detalles específicos sobre cada error para facilitar su reproducción y corrección, contribuyendo a mejorar la calidad del sitio.   

Elementos Clave Analizados 🔍

 * Reporte Detallado de Bugs 📝: Se creó un reporte para cada bug, incluyendo información como título, ID, caso de prueba, módulo/sección, prioridad, entorno, sistema operativo, descripción, datos de prueba, pasos para reproducir, resultado esperado y resultado obtenido..
   
 * Análisis de Campos Específicos 🕵️‍♂️: Se analizaron campos específicos del sitio web de Nike, como el campo "Date of Birth" en la información de la cuenta y los campos de dirección de entrega (nombre, apellido, dirección).
   
 * Clasificación de Prioridad ⭐️: Se asignó una prioridad a cada error para indicar su impacto y urgencia de resolución..
   
 * Documentación de Pasos de Reproducción 👣: Se documentaron los pasos exactos para reproducir cada error, permitiendo a los desarrolladores comprender y corregir el problema.
      
Hallazgos Clave 📌

* Se encontraron errores relacionados con la validación de datos en campos como la fecha de nacimiento y la información de la dirección de entrega..   
* Se identificaron problemas que permitían el ingreso de datos inválidos, como fechas de nacimiento incorrectas y caracteres especiales en campos de nombre y dirección..
